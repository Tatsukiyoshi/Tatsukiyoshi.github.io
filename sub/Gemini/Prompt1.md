# 開発・技術タスク特化型 依頼テンプレート（実例）

## 1. 目的と概要
* **目的:** ReactとVue.jsのフレームワーク選定のための比較。
* **利用用途:** チーム開発ドキュメント（GitHub Pagesで公開）。

## 2. 依頼タスクの詳細
* **タスクの種類:** 技術の比較分析。
* **対象:** React (v18以降) と Vue.js (v3以降)。
* **手順・要素:** 以下の項目で比較分析を依頼します。
    1.  学習コスト
    2.  パフォーマンス
    3.  大規模開発への適性
    4.  コミュニティの規模とエコシステム
    5.  開発者体験 (DX)

## 3. アウトプットの形式・構造の指定
* 比較内容は、上記5項目を含む**マトリックス形式（表）**で作成。
* 表の前には、各フレームワークの概要を**簡潔な段落**で解説。
* **見出し（##と###）**を使用して、セクションを明確に区切る。

## 4. 出力ファイルの条件
* **出力形式:** **Markdownファイル形式（.md）**。
* **流用先:** GitHub Pagesに直接流用できる形式。

### 💡 補足：Geminiへの依頼プロンプト例

このテンプレートの内容を基に、Geminiへ依頼する際の具体的なプロンプトは以下のようになります。
あなたは技術比較の専門家です。

依頼タスクの詳細に記載の5つの項目（学習コスト、パフォーマンス、大規模開発への適性、コミュニティの規模とエコシステム、開発者体験）について、React (v18以降) と Vue.js (v3以降) を比較分析してください。

**アウトプットの形式・条件**
1.  各フレームワークの概要を段落で記述した後、比較結果をマトリックス形式（Markdownテーブル）で作成してください。

1.  全体をMarkdownの見出し（##、###）で構成し、GitHub Pagesに流用可能なMarkdownファイル形式で出力してください。

# 実例
- https対応の開発サーバを構築する手順および起動する手順をフレームワークごとに整理してください。
- FlutterとReact Nativeでは、PWA対応のために構築・起動することを考慮してください。

##  整理対象のフレームワーク）
- Flutter
- React Native
- Astro
- Next.js
- React

##  ポイント
- Flutterは、http-serverをインストールして使う
- 共通性がある項目については、比較できるようにマトリックスを活用してください。
- 手順については、macOSとWindowsそれぞれの手順（可能な範囲でBunを使う）を示してください。
  - なお、WindowsはPowerShellで実行する手順としてください。
- 整理した結果をMarkdownの見出し（##、###）で構成し、GitHub Pagesに流用可能なMarkdownファイル形式で出力してください。
  また、結果をプレビューしてください。
